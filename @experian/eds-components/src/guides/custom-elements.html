HTML Custom Elements

EDS components are nothing more than custom html elements. We've tried to keep the "framework" aspects of this project to a minimum, and where possible we seek to mirror native apis and adhere to the spirit of the native web platform. As a result, there is a lot of documentation elsewhere that you can read to familiarize yourself with this project. Before diving into code, we suggest reading through the guides listed below and getting comfortable with the concepts they contain.

  • Custom Elements v1: Resuable Web Components (Google)
  • Using Custom Elements (MDN)
  • Using Shadow DOM (MDN)
  • Using Templates and Slots (MDN)

Webcomponents.js Polyfill

Custom elements are not natively available in all browsers, but polyfills make them possible to use today. EDS components make use of the webcomponents.js polyfill for a subset of shadow DOM support known as ShadyDOM. This polyfill is forced, even in browsers that support shadow DOM natively, to ensure we have consistent behavior across all platforms.

It's important to note that while we make use of shadow DOM apis for custom element composition, we are NOT using anything related to shadow DOM styling. Styling the shadow DOM requires css custom properties and we've chosen not to use a polyfill for various technical reasons. This means, effectively, that styling is handled via normal css, though we've made some preparations for a future switch to native shadow DOM. (See the Styling Elements for more detail)
